CHANGE 1.3 - Added Signup Handler with Password Hashing (Lines 44-63)
───────────────────────────────────────────────────────────────────────
ADDED CODE:
    // Password hashing and signup handler
    async function handleSignup (req: Request, res: Response, next: NextFunction) {
      try {
        const email = req.body.email
        // Check if email is valid
        if (!validator.isEmail(email)) {
          return res.status(400).send('Invalid email')
        }

        const hashedPassword = await bcrypt.hash(req.body.password, 10)

        const user = {
          email: req.body.email,
          password: hashedPassword
        }

        const token = jwt.sign({ id: user.email }, 'secret-key')
        res.json({ token, user })
      } catch (error) {
        next(error)
      }
    }

PURPOSE:
    - Handles user registration/signup
    - Validates email format
    - Hashes password using bcrypt with 10 salt rounds
    - Creates JWT token for immediate authentication
    - Returns user data and token to client

SECURITY BENEFIT:
    - bcrypt.hash() with 10 rounds provides strong password protection
    - Even if database is compromised, passwords cannot be recovered
    - JWT token enables stateless authentication
    - Error handling prevents information disclosure

                        4. SECURITY BENEFITS SUMMARY
================================================================================

EMAIL VALIDATION (validator.js)
──────────────────────────────
Security Benefit: Input Protection
- Ensures only valid email addresses are accepted
- Prevents database injection through malformed emails
- Reduces data quality issues
- RFC 5322 compliant email format validation

IMPLEMENTATION:
    validator.isEmail(req.body.email)

PROTECTION AGAINST:
    • Email format injection
    • Invalid data storage
    • Email-based enumeration attacks


PASSWORD HASHING (bcrypt)
─────────────────────────
Security Benefit: Credential Protection
- Passwords are one-way hashed, never stored in plaintext
- 10 salt rounds provide strong cryptographic protection
- Even with database breach, passwords remain secure
- Resistant to rainbow table and dictionary attacks

IMPLEMENTATION:
    bcrypt.hash(req.body.password, 10)

BENEFITS:
    • Passwords unrecoverable from database
    • Each password hashed with unique salt
    • Computationally expensive to crack
    • Industry standard for password storage


JWT AUTHENTICATION (jsonwebtoken)
──────────────────────────────────
Security Benefit: Session Management
- Stateless authentication (no server-side session storage)
- Can be verified offline without database lookup
- Prevents session fixation attacks
- Token expiration provides automatic logout

IMPLEMENTATION:
    jwt.sign({ id: user.email }, 'secret-key')

SECURITY FEATURES:
    • Generated with secret key (cannot forge)
    • Includes user identification
    • Can have expiration time
    • Cannot be modified without detection
    • Enables microservices authentication


SECURITY LOGGING (Winston File Transport)
──────────────────────────────────────────
Security Benefit: Audit Trail & Forensics
- Persistent record of all application events
- Enables post-incident analysis
- Detects suspicious patterns
- Supports regulatory compliance

IMPLEMENTATION:
    new winston.transports.File({ filename: 'security.log' })

USE CASES:
    • Detect brute force login attempts
    • Track unauthorized access attempts
    • Identify suspicious user behavior
    • Compliance reporting (PCI-DSS, HIPAA, SOC2)
    • Forensic investigation support


HELMET SECURITY HEADERS
──────────────────────
Security Benefit: Multiple Attack Vector Protection
- Sets HTTP headers to protect against common attacks
- Browser enforces security policies
- No code changes needed, headers handle protection

IMPLEMENTATION:
    app.use(helmet())

PROTECTION AGAINST:
    • MIME type sniffing attacks
    • Clickjacking/UI redressing
    • XSS (Cross-Site Scripting)
    • Man-in-the-middle attacks
    • Unauthorized frame embedding
    • DNS prefetching attacks

HEADERS SET:
    ✓ X-Content-Type-Options: nosniff
    ✓ X-Frame-Options: DENY
